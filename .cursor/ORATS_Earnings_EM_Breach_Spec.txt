VERSION: v1.0
PURPOSE:
Build a small web app where a user enters a US equity ticker and gets:
(1) breach rate (%) over the last 20 earnings (cap to ~5 years)
(2) “average above breach %” (how far beyond implied it tends to go, conditional on breach)
(3) a table of the last 20 earnings with all intermediate numbers

HIGH-LEVEL APPROACH:
- Use ORATS Delayed Data API (v2) for:
  - earnings history: /datav2/hist/earnings
  - historical “core” snapshot (for implied earnings move): /datav2/hist/cores
  - historical daily prices (for realized move calc): /datav2/hist/dailies
- Compute implied move from ORATS “impErnMv” (aka implied earnings move) on the correct “pricing date”
- Compute realized earnings move using daily open/close around the earnings timestamp (AMC vs BMO)
- Compute breach = abs(realized_move_pct) > abs(implied_move_pct) * breach_multiple (default 1.0)

WHY THESE ORATS FIELDS / ENDPOINTS:
- ORATS defines “earnings moves studies” using implied earnings moves and actual post-earnings moves (ernMv*, impErnMv). We’ll use impErnMv and compute realized ourselves to reach 20 earnings (ORATS publishes ernMv1..ernMv12, but we want 20). :contentReference[oaicite:0]{index=0}
- Delayed Data API base URL is https://api.orats.io/datav2 and uses token query param. :contentReference[oaicite:1]{index=1}
- Request limit is 1000 requests per minute (plan batching/caching). :contentReference[oaicite:2]{index=2}
- /datav2/hist/earnings exists and returns earnings history. :contentReference[oaicite:3]{index=3}
- /datav2/hist/cores exists and supports requesting fields. :contentReference[oaicite:4]{index=4}
- /datav2/hist/dailies exists for daily open/close, etc. :contentReference[oaicite:5]{index=5}

-----------------------------------------
BUSINESS LOGIC
-----------------------------------------

INPUTS:
- ticker (string, uppercase)
- lookback_earnings = 20
- max_years = 5 (filter earnings events to last ~5y)
- breach_multiple (float, default 1.0; later expose 1.0/1.5/2.0 toggles)

STEP 1 — Get earnings events
- Call /datav2/hist/earnings?ticker=XYZ
- Sort by earnDate descending
- Filter to last max_years (approx by date)
- Take first lookback_earnings events (20)
- If fewer than 20 exist, use whatever is available and display “N events found”.

STEP 2 — Determine event type and pricing date
We need “pricing date” = the date we grab implied earnings move (impErnMv).

Rules:
- If anncTod indicates “after close” (AMC):
    pricing_date = earnDate
    realized window = close(earnDate) -> open(next trading day)
- If anncTod indicates “before open” (BMO):
    pricing_date = prior trading day to earnDate
    realized window = close(prior trading day) -> open(earnDate)
- If anncTod is unknown:
    fallback to close(prior) -> open(next) around earnDate OR mark event “unknown timing” and skip in breach calc (but still show row)

NOTE:
We will not assume weekends/holidays; we must find the nearest prior/next trading day by probing /hist/dailies for dates until data exists.

STEP 3 — Pull implied move (impErnMv) for each pricing_date
For each event:
- Call /datav2/hist/cores?ticker=XYZ&tradeDate=pricing_date&fields=ticker,tradeDate,stockPrice,impErnMv
- implied_move_pct = abs(impErnMv)
- If cores data is missing for that date, retry with nearest prior trading day (max 5 attempts) and record actual used date.

STEP 4 — Pull realized move using dailies
For each event, get the needed dailies:
- For AMC:
    A = dailies(earnDate).clsPx
    B = dailies(next_trading_day).open
    realized_move_pct = abs(B - A) / A
- For BMO:
    A = dailies(prior_trading_day).clsPx
    B = dailies(earnDate).open
    realized_move_pct = abs(B - A) / A

Store:
- ref_close_date_used (the date for A)
- ref_open_date_used (the date for B)
- ref_close_px (A), ref_open_px (B)

STEP 5 — Breach flag + “above breach %”
For each event where we have implied + realized:
- breach = realized_move_pct > implied_move_pct * breach_multiple

If breach:
- above_breach_pct = (realized_move_pct - implied_move_pct) / implied_move_pct
  (example: implied 5%, realized 7% => above_breach_pct = 40%)

Also store:
- implied_move_pct
- realized_move_pct
- ratio_realized_to_implied = realized_move_pct / implied_move_pct

STEP 6 — Summary outputs for UI
Compute across valid events:
- breach_rate = mean(breach) * 100
- avg_above_breach_pct = mean(above_breach_pct for breached events) * 100
- avg_realized_pct_if_breach = mean(realized_move_pct for breached events) * 100
- avg_realized_pct_all = mean(realized_move_pct) * 100
- avg_implied_pct_all = mean(implied_move_pct) * 100
- count_events, count_valid_events, count_breaches

DISPLAY NOTES:
- Show how many events were usable vs skipped
- Show which events were skipped and why (missing core data / missing dailies / unknown timing)

-----------------------------------------
APP ARCHITECTURE
-----------------------------------------

Frontend (HTML in browser):
- Single page: ticker input, submit button
- Results area:
  - “Breach Rate (last 20 earnings)”
  - “Avg Above Breach % (conditional)”
  - (optional) toggle breach_multiple (1.0 / 1.5 / 2.0)
  - Table of events (last 20):
      earnDate | anncTod | type (AMC/BMO/UNK)
      pricing_date_used | impErnMv (%)
      close_date_used | close_px
      open_date_used | open_px
      realized_move (%)
      breach (Y/N)
      above_breach (%)
- Basic styling (clean, readable); no heavy frameworks required.

Backend (recommended):
- Python FastAPI (or Flask) providing:
  GET /api/breach?ticker=XYZ&n=20&years=5&k=1.0

Why backend:
- Keep ORATS token secret (don’t embed token in frontend JS)
- Add caching + retry logic
- Normalize and return clean JSON

Caching:
- In-memory cache keyed by (ticker, n, years, k) with TTL (e.g., 6 hours)
- Also cache ORATS raw responses by (endpoint, ticker, date) with TTL to reduce request volume
- Respect ORATS 1000 req/min limit by:
  - small concurrency
  - exponential backoff on 429/rate-limit

Errors / validation:
- Validate ticker as uppercase A-Z, 1–6 chars (allow dot/dash optionally if needed)
- Handle ORATS API errors gracefully and surface message to UI
- If no earnings history returned, show “No earnings data found.”
